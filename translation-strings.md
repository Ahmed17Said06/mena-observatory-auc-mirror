# MENA Observatory — Translation Strings
**For translation team use.**
- **Column A** = translation key or location identifier (do not translate)
- **Column B** = English text (source)
- **Column C** = Arabic translation (fill in blank cells)

There are two types of strings:
1. **`@lang` keys** — stored in `resources/lang/en/translation.php` and `resources/lang/ar/translation.php`. These are already wired into the site and only need the Arabic cell filled in.
2. **Hardcoded page strings** — text that is written directly in the page template and will need a developer to wire up after translation.

## Notes for Translation Team

- **Bold** and *italic* formatting in the "Hardcoded" section shows text that will appear bold/italic on screen — keep the same emphasis in Arabic.
- The `privacy` field contains HTML — translate only the visible words (PRIVACY - TERMS), keep the HTML tags intact.
- The `mena-ai-bbservatory` field contains an HTML line break (`<br>`) — keep it in the same position in the Arabic translation.
- Dates in the Our Story timeline (Feb 2010, Feb 2024, Dec 2024) should be kept in the same format or localized per Arabic convention.
- All other @lang fields are plain text.
